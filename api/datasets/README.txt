This folder will hold the API routes and services to use the integration_datasets registry.
Endpoints (to be implemented):
- GET /api/datasets/:key/consult?document=...
- GET /api/consultations/history?tenant_id=...&page=...&dataset=...&document=...&from=...&to=...
- GET /api/consultations/aggregations/time-series?dataset=...&from=...&to=...
- GET /api/consultations/aggregations/by-cnpj?dataset=...&from=...&to=...
- POST /api/wallets/:wallet_id/import (CSV/XLSX one-column)

The backend will:
- Resolve dataset config by key from public.integration_datasets
- Build URL/body using the document and templates
- Call provider and persist into public.consultations and public.processes
